# Demo and GIF Plan

## Goal

AetherOS needs visual proof of progress to improve educational value and GitHub appeal.

## Demo Priorities

1. Boot serial log
2. VGA text output
3. Keyboard input echo
4. Timer tick or interrupt activity
5. Memory map or allocator walkthrough

## Recommended Assets

- short GIF for README
- longer terminal capture for releases
- static screenshot in `assets/screenshots/`
- architecture diagram in `assets/diagrams/`

## GIF Guidelines

- keep them under 15 seconds
- show one concept only
- add a short caption
- prefer crisp terminal output over noisy full-screen captures

## README Placement

Use one high-value GIF near the top of the README once Boot or VGA is visually demonstrable.

## Release Placement

Each release should include:

- one hero image or GIF
- one technical summary
- one next-step section
