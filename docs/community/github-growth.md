# GitHub Growth Strategy

## Positioning

AetherOS should be presented as:

- a serious Rust OSDev project
- an educational reference
- a contributor-friendly repository

The combination matters. Many repositories have one of these strengths but not all three.

## What Attracts Stars

The highest-impact star drivers for this project are:

- a strong README with visual proof of progress
- visible roadmap discipline
- clear subsystem progression
- educational tutorials
- consistent releases with screenshots or GIFs

## What Reduces Attraction

- no visible runtime output
- no screenshots or demos
- unclear entry point for contributors
- docs that feel like placeholders
- inconsistent GitHub hygiene

## Recommended Growth Actions

1. Add a boot log GIF or asciinema capture once QEMU execution is wired.
2. Publish milestone-based releases with screenshots and learning notes.
3. Keep the README updated with the exact current subsystem stage.
4. Use Discussions for architecture questions and learning threads.
5. Convert good issues into tutorial-grade documentation after resolution.

## Suggested Discussion Categories

- Architecture
- Boot and Bring-up
- Beginner Questions
- Roadmap
- Show and Tell

## Suggested Release Pattern

Each release should include:

- milestone summary
- what was implemented
- what was intentionally deferred
- screenshots or terminal captures
- what contributors can work on next
